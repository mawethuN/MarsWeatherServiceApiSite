Critical Reflection- Proposed UI/UX

INTRODUCTION

User Interface and User Experience are the key components that shape our communication within the digital world. UX/UI often dictate the way in which we are to respond to a product. These interactions are extremely crucial, especially when designing front end products to be displayed on screens. Everything must be designed in a way as to not exclude anyone while making the experience enjoyable. In this article I will be expanding on my design process and how I went about conducting my UI/UX design. I will do so by highlighting the principles of UI/UX that have been discussed within this degree, along with additional readings related to recent weeks as they too also focus the importance of Data Visualisation and how to properly style data on a website.

STORYTELLING WITH DATA

Although I have not yet arrived at the part whereby, I implement Nasa data into my website, I have read the Knaflic reading prescribed in week 4 which mainly focused on design being used to grab the attention of users (Knaflic, 2015). The reading spoke about the importance of using UI to make sure the user gravitates towards what’s most important on the screen. This reading also touches on memory, and describes 3 types of memory that developers and UI designers must take note of. These 3 types of memory include short-term memory, iconic memory, and long term memory. According to Knaflic, iconic memory is subconscious level memory and is mainly dictated by the environment you are in. Iconic memory is able to subconsciously view large quantities of information while subconsciously storing it and processing it into short-term memory (Knaflic, 2015). Short term memory is the somewhat temporary stage of digesting information whereby the user will most probably remember one thing from an entire collection of information. Usually that one thing can be associated with colour which has its own section of connotations that subconsciously dictate what the users should pay attention to. The more important something is, the more interested we become in learning more about it which develops into long term memory. Here we find that the fragment of information consumed by the user lasts longer in their memory to a point where certain word could trigger a response related to that information. Why is this necessary or important for this article? Well to put it shortly, Knaflic goes on to further elaborate the ways in which we can use this knowledge of memories to further better the user interface of what we are developing. Knaflic suggests using pre-attentive attributes when developing or creating websites may be one of the key things to guarantee UI/UX success. Knaflic has other principles that have been stated in different chapters that helped to further develop this website.

DESIGN PROCESS AND WIREFRAMES

For this assignment, I decided to revisit the site I did last semester. I read through all the feedback and compared it to other websites. Since this semester we will be using Nasa’s API, I decided to do a weather-based website that uses an open API from NASA called INSIGHT: MARS WEATHER SERVICE. This API shows the data collected on Mars, in an area called Elysium Planitia near the equator. The data consists of temperature, wind, and pressure measurements. I created a template of what the website will look like when finished. This is what I hope to achieve by the time I am done with this website. I used the website called FIGMA to create these wireframes. Each frame or picture represents a different page on the website.
